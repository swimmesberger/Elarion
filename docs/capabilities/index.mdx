[Concepts](/docs/concepts) covers the application model you cannot avoid: every Elarion app has
handlers, results, modules, and the source generation that wires them. **Capabilities are different —
they are the platform pieces you opt into and explicitly wire.** You can ship a working application
without any of them, and add each one only when a real requirement calls for it.

That line is the framework's guiding principle in practice:

> Auto-detect application patterns, explicitly wire platform capabilities.

Each capability below is independent. Add its package, declare any compile-time opt-in it needs, and
wire the runtime in your host.
