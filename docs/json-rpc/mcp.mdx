---
title: MCP server
description: Expose your JSON-RPC handlers as a Model Context Protocol (MCP) server, independent of the JSON-RPC HTTP endpoint.
---

`Elarion.AspNetCore.Mcp` exposes the same `[RpcMethod]` handlers you already write as tools on an
[MCP](https://modelcontextprotocol.io) server, over Streamable HTTP. It is the only Elarion package that
references the `ModelContextProtocol` SDK, so plain JSON-RPC hosts never pull it in.

Nothing about a handler changes: a handler participates as a tool by virtue of its existing `[RpcMethod]`
attribute. The MCP surface is an independent endpoint over the same dispatcher — you do **not** have to
expose the public JSON-RPC endpoint to use it.

## Package reference

```xml
<ItemGroup>
  <PackageReference Include="Elarion.AspNetCore.Mcp" Version="0.1.0" />
</ItemGroup>
```

## Setup

```csharp
using Elarion.AspNetCore;
using Elarion.AspNetCore.Mcp;

var serializerOptions = CreateSerializerOptions(builder.Configuration);

// Registers JSON-RPC options and the frozen JsonRpcDispatcher singleton in one call.
builder.Services.AddJsonRpc(serializerOptions, RpcMethodMap.RegisterAll);

// Registers the MCP server with one tool per [RpcMethod] handler.
builder.Services.AddElarionMcp(RpcMethodMap.McpMetadata(), serializerOptions, o => o.ServerName = "MyApp");

var app = builder.Build();

app.MapElarionMcp();   // mounts /mcp — calling MapJsonRpc() is optional and independent
app.Run();
```

`RpcMethodMap.McpMetadata()` is generated alongside `RegisterAll` by the
[source generator](/docs/source-generation): a reflection-free table of tool names, descriptions, and
parameter metadata. There is no assembly scanning at runtime.

<Callout type="info">
  The MCP endpoint shares only the `JsonRpcDispatcher` singleton. `MapElarionMcp()` does not require
  `MapJsonRpc()` — you can expose MCP without exposing the public JSON-RPC endpoint at all.
</Callout>

## Tool and parameter descriptions

Descriptions come from `[System.ComponentModel.Description]`, read at compile time:

```csharp
using System.ComponentModel;
using Elarion.Abstractions;

[RpcMethod("clients.create")]
[Description("Creates a new client record.")]            // → tool description
public sealed class CreateClient(IAppDbContext db)
    : IHandler<CreateClient.Command, Result<CreateClient.Response>> {
    public sealed record Command {
        [Description("Human-readable client name.")]      // → input-schema property description
        public required string DisplayName { get; init; }
    }

    public sealed record Response(Guid Id);
}
```

The generator records the .NET property name; the JSON name is resolved at startup from your serializer's
`PropertyNamingPolicy`, so descriptions attach correctly under any naming policy.

## Controlling the MCP surface

The optional `[McpMethod]` attribute renames a tool or excludes a handler. It is purely additive — absent,
the handler is exposed with a tool name derived from its method name.

```csharp
[RpcMethod("clients.create"), McpMethod(ToolName = "create_client")]
public sealed class CreateClient : IHandler<...> { ... }

// Keep a method on JSON-RPC but off the MCP surface:
[RpcMethod("admin.purge"), McpMethod(Enabled = false)]
public sealed class PurgeEverything : IHandler<...> { ... }
```

## Options

```csharp
builder.Services.AddElarionMcp(RpcMethodMap.McpMetadata(), serializerOptions, o => {
    o.ServerName = "MyApp";                            // required
    o.ServerVersion = "1.0";
    o.EndpointPath = "/mcp";
    o.ToolNameTransform = m => m.Replace('.', '_');    // default
    o.IncludeErrorDetails = true;                      // surface JSON-RPC error code/data as structured content
});
```

`MapElarionMcp()` returns the endpoint builder, so you apply authorization yourself:

```csharp
app.MapElarionMcp().RequireAuthorization();
```

## How tool calls work

Each tool call runs through the same dispatcher as a JSON-RPC request, in its own service scope (so scoped
services such as a pooled `DbContext` are managed correctly). A successful handler result is returned as
JSON text; a failure maps to an MCP error, preserving the JSON-RPC error code and data as structured
content when `IncludeErrorDetails` is enabled.

<Callout type="warn">
  If two methods collapse to the same tool name (for example `a.b` and `a_b` under the default transform),
  tool building throws at startup, and the source generator emits an `ELMCP002` warning at build time.
  Disambiguate with `[McpMethod(ToolName = ...)]` or a custom `ToolNameTransform`.
</Callout>
